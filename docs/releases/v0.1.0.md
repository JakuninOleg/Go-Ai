# v0.1.0 release notes draft

This is the first public baseline for Go-Ai: a small OpenAI-compatible AI gateway for server-side apps.

## Highlights

- OpenAI-compatible `POST /v1/chat/completions` endpoint protected by bearer auth.
- Gemini-first routing with local model aliases.
- OpenRouter fallback for retryable upstream failures.
- Provider model discovery with in-memory refresh.
- HTTP/SSE streaming pass-through for `stream: true` requests.
- Tool-calling pass-through for OpenAI-compatible tool payloads. Apps still execute tools.
- Safe observability: request IDs, JSON stdout logs, diagnostic headers, and protected in-memory metrics.
- Docker image build support.
- Fly.io deployment configuration and GitHub Actions autodeploy workflow.
- Render deployment configuration.

## Known limits

- Go-Ai is a gateway, not an agent framework.
- It does not execute tools, manage memory, run chains, or own business authorization.
- Provider coverage is intentionally small: Gemini by default, OpenRouter as fallback/alternative route.
- Metrics are in-memory per process and reset on restart.
- Model discovery failures do not block startup; the static alias registry remains the safe baseline.
- Fallback only happens before a streaming response body is sent. Active streams are not mixed or replaced.
- There is no dashboard, billing layer, persistent cache, or enterprise multi-tenant control plane.

## Before tagging

- Merge the public-ready branch.
- Confirm CI is green, including the Docker build.
- Create the `v0.1.0` tag and GitHub release from the merged commit.
